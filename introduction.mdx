PyMuPDF is a high-performance Python library for data extraction, analysis, conversion, and manipulation of PDF and other document formats. Built as Python bindings over [MuPDF](https://mupdf.com/) — a lightweight, battle-tested rendering engine — PyMuPDF gives you fast, low-level access to document internals from clean, idiomatic Python.

## What is MuPDF?

MuPDF is a C library developed and maintained by [Artifex Software](https://artifex.com). It powers PDF rendering in many applications and is known for its speed, small footprint, and standards compliance. PyMuPDF wraps MuPDF's functionality in a Python API, so you get the performance of C without writing C code.

Both PyMuPDF and MuPDF are developed and maintained by Artifex Software.

## Supported document formats

PyMuPDF can open and work with a wide range of file types:

| Category | Formats |
|---|---|
| PDF & variants | PDF, FDF |
| eBook | EPUB, MOBI, FB2 |
| Comic book | CBZ, CBR |
| Other document | XPS, OpenXPS |
| Text & markup | TXT, SVG, HTML, XML |
| Image formats | PNG, JPEG, TIFF, BMP, GIF, PNM, and more |

For the complete list, see [Supported Formats](/resources/supported-formats).

## Key capabilities

**PDF manipulation** — PyMuPDF is the only format that supports full read-write access. You can insert, delete, copy, and reorder pages; merge and split documents; add text, images, and watermarks; encrypt and decrypt; and embed arbitrary file attachments.

**Format conversion** — Convert any supported document to PDF using `Document.convert_to_pdf()`. Convert PDF pages to images, SVG, or HTML. Generate PDFs from HTML source using the `Story` class.

**Performance** — Because PyMuPDF calls into compiled C code, it is significantly faster than pure-Python PDF libraries for most operations. Rendering, text extraction, and page manipulation are all handled natively by MuPDF.

**Minimal dependencies** — There are no mandatory external dependencies. Optional packages (Tesseract, fontTools, pymupdf-fonts) unlock additional features but are not required for core functionality.

## Importing PyMuPDF

The library is imported as `pymupdf`:

```python
import pymupdf

print(pymupdf.__doc__)
# PyMuPDF 1.x.x: Python bindings for the MuPDF 1.x.x library.
```

<Note>
  Older code may use `import fitz` — the historical import name for PyMuPDF. The `fitz` alias still works, but `import pymupdf` is preferred. See the [note on the fitz name](/quickstart#the-fitz-legacy-name) for details.
</Note>
